# ClipDict

> ⚠️ **This tool was fully generated by Claude (AI).** Use at your own risk. Code has not been hand-audited — read it before running.

A tiny local clipboard relay between your terminal / AI agent and a Chrome side panel. Push a dict of key/value pairs to a local HTTP endpoint and they show up instantly in your browser — click to copy, URLs render as clickable links.

Built for people who frequently need to move small values (URLs, IDs, tokens, names) from a terminal-based AI into their browser without the usual copy-paste gymnastics.

## How it works

```
  your script / AI  ──POST──▶  local server (127.0.0.1:52780)  ──WebSocket──▶  Chrome side panel
```

- **Server**: single-file `aiohttp` app. Accepts `POST /clipboard` with a JSON dict, broadcasts to any connected WebSocket clients.
- **Extension**: Manifest V3 Chrome side panel. Connects to `ws://localhost:52780/ws`, renders items as a list. Strings matching `https?://...` become clickable links with a copy button; everything else is click-to-copy.
- **No cloud.** Everything stays on localhost.

## Install

### Server

```bash
git clone https://github.com/robertlederer27/clipdict.git
cd clipdict
pip install -r requirements.txt
python server.py
```

Listens on `127.0.0.1:52780`.

### Chrome extension

1. Open `chrome://extensions`
2. Enable **Developer mode** (top right)
3. **Load unpacked** → select the `extension/` folder
4. Pin the extension and click its icon to open the side panel

## Usage

Push some items:

```bash
curl -X POST http://127.0.0.1:52780/clipboard \
  -H "Content-Type: application/json" \
  -d '{
    "items": {
      "Invoice URL": "https://example.com/invoice/42",
      "Customer": "ACME GmbH",
      "IBAN": "AT00 0000 0000 0000 0000"
    }
  }'
```

The side panel updates instantly. Click any text value to copy it. Click a link to open it; click the 📋 icon next to a link to copy it instead.

### From Python

```python
import requests
requests.post("http://127.0.0.1:52780/clipboard", json={
    "items": {
        "Docs": "https://docs.python.org/3/",
        "Token": "sk-...",
    }
})
```

### From an AI agent

Give your agent (Claude Code, etc.) a tool or instruction to POST to the endpoint whenever it wants to hand you a value. Perfect for "here are the 5 links you asked about" moments.

## Protocol

### `POST /clipboard`

Request body:
```json
{ "items": { "<label>": "<value>", ... } }
```

- `items` must be a JSON object (dict).
- Values are rendered as strings. Strings matching `^https?://\S+$` become links.
- Every POST **replaces** the current state (no merging).

Response: `{"status": "ok"}`

### `GET /ws`

WebSocket. On connect, the server sends the current state. On every POST, the server broadcasts:

```json
{ "type": "update", "items": { ... } }
```

## Limitations / non-goals

- Single-user, single-machine. No auth, no TLS — it binds to `127.0.0.1` only.
- No history. New POSTs replace everything. If you want persistence, fork it.
- Not a cloud tool and never will be.
